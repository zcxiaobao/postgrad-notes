# 代码 bug 修复

本周针对三个数据集进行对比实验的测试，对比融合彩色图像，VI 轨迹，WRG 加权递归图三种特征的识别准确率。

测试过程中，发现三者区别不算特别大，但总体保持 VI < WRG < 融合图像的识别顺序。

按理说应该是符合常理的，但通过多种方式排查，总感觉存在一定问题

  1. 项目使用 10 折交叉验证，第一折的 loss 随 epoch 变化曲线非常合理，达到一定的 epoch 后收敛；但第二折则收敛非常快
  2. VI 轨迹负荷特征目前有很多文献尝试过，很少有达到 95%，因此有一定的怀疑。

查了很多资料，怀疑过过拟合现象，梯度消失现象，但经过测试都变动不大，而且测试集准确率都比较高。

经过反复阅读代码，发现问题在于每次交叉验证未重置 model，造成数据泄露，导致识别率虚高

修改代码后，重新运行各对比实验，目前只完成了一部分，根据现在完成的部分测试结果，识别准确率和 loss 曲线都变得合理。

# PLAID 2018 数据集

聚合数据集正式加入测试过程中，识别准确率融合图像取得最高，进一步证明了所提方法的泛化性能和鲁棒性。
